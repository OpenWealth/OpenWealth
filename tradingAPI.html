Download OpenAPI specification:Download

E-mail: openwealth@synpulse.com License: Apache 2.0

This API is used to place orders and check on the order status.

Find out more about OpenWealth API specifications.

account

Account operations.

Returns list of accounts the requestor has access to

Returns list of accounts the requestor has access to Paging is done based on the accountaccess object, i.e. if limit is set to 1, then 1 account object will be returned per page.

query Parameters
cursor
string

An opaque string value used for pagination.

limit
integer <int32> >= 1

Maximum number of items to return.

header Parameters
X-Correlation-ID
required
string

Unique ID (defined by the caller) which will be reflected back in the response.

Responses

Your server url

https://www.openwealth.ch/accounts

order

Order operations.

Posts a new order

Posts a new order.

header Parameters
X-Correlation-ID
required
string

Unique ID (defined by the caller) which will be reflected back in the response.

Request Body schema: application/json
clientReference
required
string <= 20 characters

Reference of the order, as assigned by the instructing party.

object (OrderStatus)

Provides the status of the order was at a certain time in its lifecycle.

side
required
string
Enum: "buy" "sell"

Side taken by a party on an order.

required
object (Quantity)

Quantity entity.

object (Quantity)

Quantity entity.

numberOfAllocations
required
number <integer>

Indicates number of orders to be combined for average pricing and allocation.

required
object (FinancialInstrument)
object (PlaceOfTrade)

Market in which a trade transaction is to be or has been executed.

currency
required
string (Currency) ^[A-Z]{3}$

ISO 4217 code.

executionType
required
string
Enum: "market" "limit" "stop" "stopLimit"

Type of the execution of the order.

object (Price)

Price entity.

object (Price)

Price entity.

timeInForce
required
string
Enum: "day" "goodTillCancel" "atTheOpening" "immediateOrCancel" "fillOrKill" "goodTillCrossing" "goodTillDate" "atTheClose" "goodThroughCrossing" "atCrossing" "goodForTime" "goodForAuction" "goodForMonth"

Specifies how long the order remains in effect.

expiryDateTime
string <date-time> (DateTime)

DateTime according to ISO 8601.

Array of objects (Allocation)
object (Quantity)

Quantity entity.

Responses

Your server url

https://www.openwealth.ch/orders

Returns all open orders to which the user of the API has access to. Paging is done based on the order object, i.e. if limit is set to 1, then 1 order will be returned per page.

cursor
string

An opaque string value used for pagination.

limit
integer <int32> >= 1

Maximum number of items to return.

header Parameters
X-Correlation-ID
required
string

Unique ID (defined by the caller) which will be reflected back in the response.

Responses

https://www.openwealth.ch/orders
