This API is used to place orders and check on the order status.

Find out more about OpenWealth API specifications.

account

Account operations.

Returns list of accounts the requestor has access to

Returns list of accounts the requestor has access to Paging is done based on the accountaccess object, i.e. if limit is set to 1, then 1 account object will be returned per page.

string

An opaque string value used for pagination.

limit
integer <int32> >= 1

Maximum number of items to return.

required

order

Order operations.

Posts a new order

Posts a new order.

query Parameters
expand
string

Expand related resources. Comma-separated values, supporting:

  • allocationList: Include allocationList details.
  • allocationList.executionAllocationList: Include executionAllocationList details on the allocation.
  • allocationList.allocationCancellationReasonList: Include allocationCancellationReasonList details on the allocation.
  • orderCancellationReasonList: Include orderCancellationReasonList
header Parameters
X-Correlation-ID
required
string

Unique ID (defined by the caller) which will be reflected back in the response.

Request Body schema: application/json
clientReference
required
string <= 128 characters

Reference of the order (unique identification), as assigned by the instructing party (i.e. TPP).

groupReference
string <= 128 characters

Optional reference to link orders, i.e. multi-leg orders.

kind
string

Kind of order requested.

  • Single leg order
  • Swap leg Order
  • Multi leg order
object (OrderStatus)

Provides the status of the order was at a certain time in its lifecycle.

bestEffortExecution
boolean
Default: false

TBD

object (PlaceOfTrade)

Market in which a trade transaction is to be or has been executed.

executionType
required
string
Enum: "market" "limit" "stop" "stopLimit"

Type of the execution of the order.

object (Price)

Price entity.

object (Price)

Price entity.

timeInForce
required
string
Enum: "day" "goodTillCancel" "atTheOpening" "immediateOrCancel" "fillOrKill" "goodTillCrossing" "goodTillDate" "atTheClose" "goodThroughCrossing" "atCrossing" "goodForTime" "goodForAuction" "goodForMonth"

Specifies how long the order remains in effect.

expiryDateTime
string <date-time> (DateTime)

DateTime according to ISO 8601.

exposureDuration
integer

This is the time in seconds of a 'goodForTime' TimeInForce. Positive integer value which represents the time is seconds in which the new order remains active in the market before it is automatically cancelled (e.g. expired).

quoteId
string <= 128 characters

Unambiguous identifier for the quote that was provided by the confirming party.

Array of objects (Allocation)
orderPlacer
string <= 128 characters

The entity physically transmitting the order to the execution venue. This role is often fulfilled by:

  • Investment firms or brokers executing trades on behalf of clients.
  • Systematic internalisers or trading venues handling order transmission.
  • In some cases, the order placer may also be the decision maker if no third-party authorization exists.
decisionMaker
string <= 128 characters

The party legally authorized to make investment decisions on behalf of a client. This typically includes:

  • Individuals or entities acting under a power of attorney (e.g., attorneys managing accounts).
  • Discretionary investment managers executing trades without prior client approval.
  • In cases like funds, the fund manager is generally considered the decision maker.
object (AlgoTrade)
additionalDetails
string <= 512 characters

Provides additional details on the order which can not be included within the structured fields of the message.

required
object (OrderLeg)

Order object.

Responses

Your server url

https://www.openwealth.ch/orders

Returns a list of open orders

Returns all open orders to which the user of the API has access to. Paging is done based on the order object, i.e. if limit is set to 1, then 1 order will be returned per page.

cursor
string

An opaque string value used for pagination.

limit
integer <int32> >= 1

Maximum number of items to return.

header Parameters

required
string

Unique ID (defined by the caller) which will be reflected back in the response.

Responses
